# Security Policy

## Supported Versions

Security is one of Prism's core goals, all security vulnerabilities or suspected security vulnerabilities should be reported privately, so vulnerabilities will be investigated and patched to mimize risk. 
Any information could be kept entirely internal.

You will recive credit if the issue is proven real (unless you want to be annonymous), after patched releases have landed.

Please provide a working minimal test case that replicates the bug if possible.

## Reporting a Vulnerability

If you know of a publicly disclosed security vulnerability for Prism, please contact dawn@dawn.day to inform.

IMPORTANT: Do not file public issues on GitHub for security vulnerabilities.
